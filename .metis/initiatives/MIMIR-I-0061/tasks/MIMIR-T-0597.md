# Pass 1.1: Global terminology and cross-cutting fixes

## Parent Initiative

[[MIMIR-I-0061]]

## Objective

Apply cross-cutting terminology and factual fixes that affect many pages simultaneously. This task runs FIRST to establish the correct baseline before section-specific fixes begin.

## Scope

This task touches every doc page that contains any of these errors. Use grep to find all occurrences, fix them all, and verify no stragglers remain.

### Fix 1: "Object" → "Marker" (token type)

The token palette type is "Marker" (📍 icon), not "Object". Fix in all pages.

**Known affected pages:**
- `tutorials/02-first-module.md`
- `how-to/maps/place-tokens.md`
- `reference/ui/token-setup-modal.md`
- Any other page mentioning "Object" as a token type

**Verification:** Check `TokenPalette.vue` — the type enum should confirm "Marker" (was line 276 at audit time).

### Fix 2: "4 tabs" → "5 tabs" (dashboard)

The campaign dashboard has 5 tabs: Campaign, Modules, NPCs, PCs, **Homebrew**. All references to "four tabs" or listing only 4 must be updated.

**Known affected pages:**
- `tutorials/01-first-campaign.md`
- `reference/ui/campaign-dashboard.md`
- `reference/glossary.md` (Campaign Dashboard definition)
- `explanation/two-board-system.md`

**Verification:** Check `useDashboardState.ts` — `dashboardTabs` array should have 5 entries.

### Fix 3: "Dangers" → "Monsters" (module prep)

Module prep uses `ModuleMonsters.vue`, labeled "Monsters" not "Dangers".

**Known affected pages:**
- `reference/ui/module-prep-view.md`
- `explanation/two-board-system.md`

**Verification:** Check `ModuleMonsters.vue` component name and any label text.

### Fix 4: Glossary additions

Add missing terms to `reference/glossary.md`:
- **Homebrew** — Custom content (items, monsters, spells) created by the DM within a campaign
- **Sidecar** — The MCP server binary that runs alongside Mimir, enabling AI-assisted campaign management
- **MCP (Model Context Protocol)** — Protocol used by the Mimir sidecar to expose campaign management tools to AI assistants
- **Clone from Catalog** — Feature to create homebrew content by copying and modifying an existing catalog entry
- **Mapgen** — Procedural map generation system that produces Dungeondraft-compatible map files

### Fix 5: File formats additions

Add to `reference/file-formats.md`:
- `.dungeondraft_map` — Dungeondraft native map format (JSON), generated by mimir-mapgen
- Clarify `.mimir-campaign.tar.gz` as the specific campaign archive extension
